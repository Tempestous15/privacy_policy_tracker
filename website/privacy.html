Privacy Policy

Last updated: July 2026

ClipPri has no server and no accounts. The browser extension does all of its work locally, on your device. This policy explains exactly what that means in practice.

We don't operate a backend

There is no ClipPri server. We don't collect, store, or have access to the sites you visit, the policies you scan, or anything the extension does on your behalf. There's nothing for us to see, because nothing is sent to us.

What the extension does locally

  • Reads the current page, and follows its privacy-policy link wherever it leads — often a different domain than the site itself (e.g. a company's policy hosted on a separate legal/policies subdomain). The extension requests broad page-access permission specifically to make this work; it's used only to locate and fetch that policy text, which never leaves your device.
  • Scans that text on-device for red flags (data selling, vague third-party sharing, indefinite retention, arbitration clauses, and similar patterns) using a local, non-AI classifier bundled with the extension.
  • Generates plain-English summaries on-device using a small language model that runs in your browser via WebGPU. The policy text you're summarizing is never transmitted anywhere to produce that summary.
  • Saves sites you choose to save in your browser's own local extension storage. This list lives only on your device and is never synced to us.
  • Watches which third-party domains a site contacts as you browse (using the browser's webRequest API) and checks them against a tracker-categorization dataset bundled with the extension, from DuckDuckGo's Tracker Radar. This only observes requests your browser is already making — it doesn't modify, block, or add any network traffic. What's captured for a tab is kept in the browser's own temporary session storage, cleared automatically when that browsing session ends, and is never transmitted anywhere.

About the "read and change data on all websites" permission

The extension requests broad page/network access (webRequest and broad host_permissions) for two reasons covered above: following a privacy policy link to whatever domain it lives on, and observing third-party requests for the tracker-detection feature. Despite the permission's name, the extension cannot see the content of your browsing (page contents on other tabs, passwords, form data) through this permission — it only reads request metadata (which domains were contacted) and page text you explicitly asked it to check.

The one third-party request the extension makes

To show community-reviewed ratings for well-known sites, the extension queries the public ToS;DR API directly from your browser. That request goes to ToS;DR, not to us — see ToS;DR's own privacy policy for how they handle it.

The first time the on-device summary feature is used, your browser also downloads the language model's weights from a model-hosting CDN. That's a download of static model files, not a request containing any policy text or browsing data.

The website: scan history and URL check

Two pages on this website talk to data outside this static site's own files, and it's worth being precise about exactly what that means.

  • Scan History asks the ClipPri extension, if it's installed, for the list of sites you've saved -- using Chrome's externally_connectable messaging API. This is a same-device, browser-internal message between this page's JavaScript and the extension running in the same browser; it is not a network request, and nothing about it reaches any server, including ours. The extension only accepts this message from this website's own address (configured in the extension's manifest.json) -- no other website can ask it for this.
  • Check a URL makes a direct request from your browser to the public ToS;DR API for a community rating (same endpoint and same "goes to ToS;DR, not to us" note as the extension -- see above), and looks up the domain you typed against a small bundled dataset shipped as a static file with this website. Neither of those touches a server of ours. This page cannot fetch and scan an arbitrary site's actual policy text the way the extension does -- that requires a permission only an installed browser extension can hold, so this page's local-classifier scan is limited to sites already in the bundled dataset. See the page itself for the exact scope.

Neither page adds a server, a database, or any logging on our side -- this website is still exactly what "What we don't do" (below) describes. If a future version needs a server-side component for either feature (for example, to work around a browser restriction), that would be a real architecture change, and this policy will say so explicitly before it ships.

What we don't do

  • No accounts, no sign-up, no login
  • No analytics or usage tracking
  • No server-side storage of anything you scan or save
  • We never see your browsing history or the content of pages you visit
